1.Redis相关

发表于 2016-06-24   |  

1.简介

开源免费,遵循BSD()协议,是一个高性能的key-value数据库

特点:

  1. 支持数据的持久化
  2. 支持list,set,zset,hash等数据结构
  3. 支持Master-slave模式的数据备份

优点:

高性能;数据类型丰富;所有操作是原子的,支持丰富的特性

2.安装方式

使用homebrew在mac上安装,使用brew services start redis开启服务;

3.基本数据类型

String(字符串)

二进制安全的,可以包含任何数据(一个键最大为512M)

区分大小写

Hash(哈希)

Redis中的hash是一个键值对集合

包含一个String类型的field和value的映射表

实例中我们使用了 Redis HMSET, HGETALL 命令，user:1 为键值。

每个 hash 可以存储 2^(32 - 1) 键值对（40多亿）。

List(列表)

Redis中的列表是简单的字符串列表,按插入顺序排序,可以从头或者尾插入元素

可以存储 2^(32 - 1) 键值对（40多亿）。

操作方法 (lpush aa )

Set(集合)

通过哈希表实现,达到O(1)的复杂度

sadd key member

注意：以上实例中 rabitmq 添加了两次，但根据集合内元素的唯一性，第二次插入的元素将被忽略。

集合中最大的成员数为 2^(32 - 1) (4294967295, 每个集合可存储40多亿个成员)。

zset(有序集合)

不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。

zset的成员是唯一的,但分数(score)却可以重复。

zadd key score member
